## Git Commit Conventions

Commits follow the format: `<type>: <short description>`

### Types
`chore:` build, dependencies, configuration
`feat:` new feature
`style:` formatting with no logic impact
`refactor:` code restructuring without behaviour change
`fix:` bug fix
`docs:` documentation

### Rules
Description in lowercase and no trailing period.
Use imperative mood: add, fix not added, fixes.
